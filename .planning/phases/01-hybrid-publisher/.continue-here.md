**The authentication journey**:

We discovered that the publisher endpoint was using `HTTPBearer()` (expects `Authorization: Bearer <token>` header) but OAuth uses httpOnly cookies. So we created `get_current_auth_user_from_cookie` that:

1. Reads `access_token` from Request.cookies
2. Verifies with JWT service
3. Fetches User from DB
4. Returns User entity

**But it's still failing with 401**. The SQL query runs (we see the user SELECT in logs) but then ROLLBACK happens. This suggests:
- The user IS found in the DB
- But something AFTER the user lookup is causing the exception
-可能是 user.tenant_id is None (multi-tenancy requirement)?
- Or there's a middleware/exception handler catching something

**The weird part**: Debug logs aren't showing. Either:
1. Logging level is too high (need to set to INFO/DEBUG)
2. The exception happens before logging
3. Docker logs are filtered
