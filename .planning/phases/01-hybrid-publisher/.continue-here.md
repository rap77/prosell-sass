Después de ~7 intentos con diferentes patrones CSS (flexbox hacks, calc(), min(), shadcn patterns), systematic debugging reveló que el problema real NO era el CSS sino un error de TypeScript que previno la compilación.

**Root cause**: `tenant_id: number` en PublishModal vs `tenant_id: string` en catalog/page.tsx

**Solución**: Cambiar tipo a `string` en PublishModal + rebuild completo

**Patrón ganador** (el más simple posible):
- Dialog.Content: `rounded-xl overflow-hidden max-h-[85vh] flex flex-col`
- Content: `overflow-y-auto flex-1 min-h-0`
- Header: `flex-shrink-0` (sticky)
- Footer spacer: `h-6 flex-shrink-0`

No se necesitan cálculos complejos, `Dialog.Description`, ni envoltorios adicionales.
