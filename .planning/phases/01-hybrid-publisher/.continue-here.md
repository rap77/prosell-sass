### Session 2026-03-24: Complete Root Cause Analysis & Fixes

#### 1. Auth 401 Unauthorized ✅ FIXED
**Root Cause**: `get_publish_vehicle_use_case` injected `get_current_auth_user` (Bearer token) but endpoint used `get_current_auth_user_from_cookie` (cookies). FastAPI executed BOTH dependencies.

**Fix**: `apps/api/src/prosell/infrastructure/api/dependencies.py` line 657
```python
# Changed from:
current_user: Annotated[User, Depends(get_current_auth_user)],
# To:
current_user: Annotated[User, Depends(get_current_auth_user_from_cookie)],
```

#### 2. UUID Validation 422 ✅ FIXED
**Root Cause**: Mock data used string IDs like `"veh-001"`, `"page-001"` but backend validates UUIDs.

**Fixes Applied**:
- `apps/web/src/app/dashboard/catalog/page.tsx`:
  - Vehicle IDs: `"01234567-89ab-cdef-0123-456789abcdef"` (Toyota Corolla)
  - Vehicle IDs: `"12345678-9abc-def0-1234-56789abcdef0"` (Honda Civic)
  - Vehicle IDs: `"34567890-bcde-f012-3456-789abcdef012"` (Ford Focus)
  - Publication IDs: Also updated to UUIDs
  - Facebook Pages: `"aaaaaaaa-aaaa-aaaa-aaaa-000000000001"`, etc.

#### 3. Vehicle ID Missing From vehicleData ✅ FIXED
**Root Cause**: `PublishForm` received `vehicleData` object without the `id` field. The catalog's `selectedVehicle` transformation didn't include `id`.

**Fixes Applied**:
- `apps/web/src/app/dashboard/catalog/page.tsx` line 188: Added `id: selectedVehicleRaw.id`
- `apps/web/src/components/publisher/PublishForm.tsx`: Updated `VehicleData` interface to include `id: string`
- `apps/web/src/components/publisher/PublishModal.tsx`: Updated `VehicleData` interface to include `id: string`
- `apps/web/src/components/publisher/PublishForm.tsx` line 237: Changed to `product_id: vehicleData.id || vehicleData.tenant_id`

#### 4. CORS Policy Error ✅ FIXED
**Root Cause**: API bound to `127.0.0.1:8000` but frontend calls `localhost:8000`. For CORS, these are DIFFERENT origins.

**Fixes Applied**:
- Restarted API with `--host 0.0.0.0` instead of default `127.0.0.1`
- `apps/api/.env`: Fixed ALLOWED_ORIGINS format from `["http://localhost:3000","http://localhost:8000"]` to `http://localhost:3000,http://localhost:8000`
- Verified CORS works: `curl -H "Origin: http://localhost:3000" -X OPTIONS http://localhost:8000/api/v1/publisher/test/publish -v` returns `access-control-allow-origin: http://localhost:3000`

<decisions_made>

1. **API Binding**: Use `--host 0.0.0.0` for local development so both `localhost` and `127.0.0.1` work
   - **Rationale**: Browsers treat `localhost` and `127.0.0.1` as different origins for CORS

2. **Mock Data Format**: Always use proper UUID format, never `"test-001"` style strings
   - **Rationale**: Backend validates UUIDs; string IDs cause 422 validation errors

3. **Auth Dependencies**: Publisher endpoints use cookie auth (`get_current_auth_user_from_cookie`), not Bearer tokens
   - **Rationale**: OAuth flow sets httpOnly cookies; Bearer tokens require manual header setting

4. **Environment Format**: Pydantic CSV parsing needs `val1,val2` not `["val1","val2"]`
   - **Rationale**: The `[]` and `""` characters break the split logic, creating invalid origins

5. **Vehicle Data Flow**: Vehicle ID must be included in `vehicleData` object passed to `PublishForm`
   - **Rationale**: The form uses `vehicleData.id` for `product_id`; without it, validation fails

</decisions_made>
