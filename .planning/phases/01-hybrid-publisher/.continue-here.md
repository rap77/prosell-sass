<decisions_made>

**Trigger de publicación:**
- Modal desde el catálogo (overlay, no navega fuera)
- 6 campos: título, descripción, precio+delta CarGurus, FB Page selector, Hero Shot (click=first), ZIP Code (pre-filled dealer)
- Botón contextual: "Publicar" → "Actualizar" cuando ya hay listing activo (muestra diff de precio)
- Validación Zod en frontend antes de llegar a Playwright

**Manejo de errores (3 categorías):**
- A (Transient): exponential backoff automático (1m→5m→15m), 3 retries, luego "Atención Requerida"
- B (Bloqueante/captcha): pausa cola del vendedor, retry bloqueado hasta checkbox "Ya validé mi cuenta"
- C (Validación): bloqueado en frontend, nunca ejecuta Playwright
- Visibilidad: badge en catálogo + detalle en modal + widget "Pending Actions" en dashboard vendedor
- Admin: vista global de fallos cross-vendedor para detectar patrones

**Selector de estrategia:**
- Feature flag `PUBLISHER_ENGINE=playwright|graph_api|auto`
- Auto en Phase 1 = Playwright primario (Graph API sin aprobación aún)
- Log por publicación: estrategia + versión del motor (`playwright_v1.42`)
- Toggle admin sin redeploy — interruptor de emergencia

**Manejo de imágenes:**
- Fuente híbrida: scraper + upload manual adicional
- Hero Shot: click para mover a índice 0 con badge "PORTADA"
- Pipeline backend: compress <1MB, resize FB estándar, convertir a JPG, strip EXIF
