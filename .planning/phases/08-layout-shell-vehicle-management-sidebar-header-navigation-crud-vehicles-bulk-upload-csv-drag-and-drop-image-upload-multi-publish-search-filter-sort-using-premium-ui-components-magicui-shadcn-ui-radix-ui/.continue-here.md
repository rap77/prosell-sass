### 1. Layout Structure ✅ COMPLETO

**Sidebar: Por Leading Indicators (Indicadores Clave)**
- Operaciones: Catálogo, Publicaciones

- Search global (Cmd+K) — Omnibar para vehículos/leads

**Mobile: Hybrid Ergonomic**
- Bottom Navigation (4 iconos críticos): Catálogo, Publicar, Leads, Más (Drawer)
- Drawer lateral para el resto (Configuración, Perfil, Logs)
- **Razón**: Reachability — acciones que generan dinero a un pulgar de distancia.

**Keyboard: Performance Shortcuts**
- Cmd+K: Command Palette (search + acciones)
- Esc: Cerrar modales / limpiar búsquedas
- Flechas + Enter: Navegación DataGrid
- **Razón**: Power user workflow — vendedores usan mucho teclado.

### 2. DataGrid Pattern ✅ COMPLETO

**Columnas: Compact (5) + Expandible**
- Foto thumbnail (60x60px redondeado)
- Título (Año/Marca/Modelo combinados)
- Precio (formato moneda local)
- Estado (Badge colorido)
- Acciones (botón contextual)
- **Expandible**: VIN, Stock, Leads, Citas en fila expandible o tooltip
- **Razón**: 5 columnas = sin scroll horizontal en laptops 13". Evita fatiga visual.

**Virtualización: TanStack Virtual (rows)**
- Buffer: 10-15 filas fuera de cámara
- 60fps scroll fluido
- **Razón**: Solo renderiza ~40 filas (20 visibles + buffer). DOM ligero.

**Selección: Checkboxes + Shift-click**
- Checkbox por fila
- Shift-click para rangos (estándar Gmail/Google Sheets)
- Indeterminate checkbox en header (Select All)
- **Floating Action Bar**: "Publicar X seleccionados" aparece al seleccionar
- **Razón**: De 2 clics a 50 seleccionados. Eficiencia de lote.

**Estados: Badges Coloridos (Semántica Visual)**
- Verde (Published): "Dinero en la mesa"
- Amarillo (Pending): "En proceso"
- Rojo (Failed): "Acción requerida"
- Gris (Draft/Expired): "Inactivo"
- **Razón**: Escaneo de alta velocidad — ojo detecta "manchas rojas" instantáneamente.

### 3. Bulk Upload Flow ✅ COMPLETO

**Validación: Zod en Frontend (Síncrona)**
- Esquema (columnas CSV coinciden con template)
- Tipos (precio es número, no string)
- Estructura (filas vacías, campos obligatorios)
- **Feedback instantáneo**: Modal se ilumina en rojo apenas suelta el CSV
- **Razón**: Backend limpio — JSON que llega a API tiene estructura correcta.

**Procesamiento: Chunks (50) + Parallel Workers**
- 3-4 workers Taskiq procesando en paralelo
- Resiliencia: Si chunk #42 falla, solo pierdes 50, no 500
- Uso de memoria predecible
- **Razón**: Un error en una imagen rota no bloquea los otros 450 autos.

**Progreso: Barra + % + ETA**
- Polling cada 2s a /bulk-upload/{job_id}/status
- Redis almacena progreso (chunks completados / total)
- ETA dinámico basado en velocidad de chunks anteriores
- **Razón**: Elimina ansiedad — "ETA: 3 min" hace que el usuario se relaje.

**Errores: Best Effort + CSV Descargable**
- Continuar procesando válidas
- Reporte final: "450 exitosos | 50 fallidos"
- CSV descargable con filas fallidas + columna `error_reason`
- **Flujo de corrección**: Descargar, corregir 50 filas, re-subir
- **Razón**: Productividad ininterrumpida — 450 autos ya están trabajando.

### 4. Role-based Navigation ✅ COMPLETO

**Route Groups: 4 Niveles**
- `(admin)`: Infraestructura (proxies, rotación FB, facturación global)
- `(dealer)`: Visión de negocio (reportes financieros, ROI, estados de cuenta)
- `(manager)`: Supervisión de equipos (DataGrid con esteroides, reasignar Leads)
- `(seller)`: Ejecución de ventas (solo sus autos, sus leads, su botón publicar)
- **Razón**: Privacidad — evita que sellers se "roben" clientes entre ellos.

**Middleware: Auth + Role + Tenant (Triple Check)**
- Autenticado (cookie/JWT existe)
- Role machea (user.role vs ruta permitida)
- Tenant scope (tenant_id del concesionario)
- **Zero Trust en Edge**: Rebota antes de que DB reciba consulta
- **Redirección inteligente**: / → middleware detecta rol → manda a su "casa"
- **Razón**: Seguridad nivel bancario + optimización de servidor.

**Fallback: Redirect + Toast**
- Re-direccionar a dashboard del rol
- Toast: "Esa sección es solo para administradores"
- **Cero fricción**: Usuario "rebota" a lugar seguro
- **Razón**: Más simple que 403 page — menos tickets de soporte.

**Role Source: JWT Token (15-30 min) + Refresh Tokens**
- Stateless — latencia cero en navegación
- Access token corto + refresh token
- **Razón**: Máxima velocidad de Middleware sin sobrecargar DB.

### 5. Image Upload UX (POR DISCUTIR)

**Pregunta clave**: ¿Presigned URLs (direct to Cloud) o via backend?

- **Presigned URLs**: Browser → Cloudinary/S3 directo, backend solo recibe URLs. Ahorra ancho de banda del servidor.
- **Via Backend**: Browser → FastAPI → Cloud. Backend procesa (resize/optimiza) antes de subir. Más control pero más carga.
- **Híbrido**: Presigned upload + backend job async para procesamiento (resize/thumbs).

**UX para múltiples fotos (10-20 por vehículo)**:
- Drag & drop zone + file picker fallback (Dropbox style)
- File picker multiple simple (más compatible móvil)
- Camera direct + gallery (mobile-first)

### 6. Search Filters (POR DISCUTIR)

**Pregunta clave**: ¿Server-side o Client-side?

- **Server-side**: URL params (/catalog?make=Toyota&price_min=10000), infinite scroll virtualizado. Escala a millones.
- **Client-side**: Sobre datos cargados (memo). Más simple pero solo escala a ~1000 filas.
- **Hybrid**: Client-side filtros rápidos + server-side rangos + debounce 300ms.

**UI de filtros**:
- Sidebar collapsible (estilo e-commerce, 30% filtros + 70% DataGrid)
- Header filters (dropdowns por columna)
- Command Palette (Cmd+K) como filtro principal
