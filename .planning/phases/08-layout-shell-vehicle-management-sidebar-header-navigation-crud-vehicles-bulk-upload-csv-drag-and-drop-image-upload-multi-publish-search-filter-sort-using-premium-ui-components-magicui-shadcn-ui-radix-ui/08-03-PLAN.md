  <name>Task 2: Create useVehicleFilters hook with URL sync</name>
  <files>apps/web/src/lib/hooks/useVehicleFilters.ts</files>
  <action>
    Create custom hook for filter state management with URL persistence:

    **State to manage:**
    - `search: string` (text search)
    - `brand: string[]` (multi-select brand filter)
    - `priceRange: [number, number]` (min/max price)
    - `status: string[]` (multi-select status filter)
    - `year: [number, number]` (min/max year)

    **Implementation with URL sync:**
    ```tsx
    'use client'

    import { useSearchParams, useRouter } from 'next/navigation'
    import { useMemo, useCallback } from 'react'

    export function useVehicleFilters() {
      const searchParams = useSearchParams()
      const router = useRouter()

      // Read from URL
      const filters = useMemo(() => ({
        search: searchParams.get('search') || '',
        brand: searchParams.get('brand')?.split(',') || [],
        priceRange: [
          Number(searchParams.get('minPrice')) || 0,
          Number(searchParams.get('maxPrice')) || 100000,
        ] as [number, number],
        status: searchParams.get('status')?.split(',') || [],
      }), [searchParams])

      // Update URL
      const setFilter = useCallback((key: string, value: string | string[]) => {
        const params = new URLSearchParams(searchParams)

        if (Array.isArray(value)) {
          if (value.length > 0) {
            params.set(key, value.join(','))
          } else {
            params.delete(key)
          }
        } else if (value) {
          params.set(key, value)
        } else {
          params.delete(key)
        }

        router.push(`?${params.toString()}`, { scroll: false })
      }, [searchParams, router])

      const clearAllFilters = useCallback(() => {
        router.push('/catalog', { scroll: false })
      }, [router])

      return {
        filters,
        setFilter,
        clearAllFilters,
      }
    }
    ```

    **Key features:**
    - URL state sync (shareable links)
    - `scroll: false` prevents page jump on filter change
    - Multi-select filters use comma-separated values
    - Clear all resets to base URL

    This hook enables both sidebar filters and Command Palette to share state.

    ```tsx
    'use client'

    import { useVehicleFilters } from '@/lib/hooks/useVehicleFilters'
    import { Checkbox } from '@/components/ui/checkbox'
    import { Label } from '@/components/ui/label'
    import { Slider } from '@/components/ui/slider'

    export function FilterSidebar() {
      const { filters, setFilter, clearAllFilters } = useVehicleFilters()

      return (
        <aside className="w-64 border-r p-4 space-y-6">
          {/* Brand filter */}
          <div>
            <h3 className="font-semibold mb-2">Brand</h3>
    ```

  <name>Task 4: Build FilterPills for active filter display</name>
  <files>apps/web/src/components/filters/FilterPills.tsx</files>
  <action>
    Create filter pills component for visual feedback above DataGrid:

    **Purpose:**
    - Show user why they see 5 cars instead of 500
    - Allow quick removal of individual filters
    - "Clear all" button for reset

    **Implementation:**
    ```tsx
    'use client'

    import { useVehicleFilters } from '@/lib/hooks/useVehicleFilters'
    import { X } from 'lucide-react'
    import { Button } from '@/components/ui/button'

    export function FilterPills() {
      const { filters, setFilter, clearAllFilters } = useVehicleFilters()

      const activeFilters = [
        ...(filters.brand.map(b => ({ key: 'brand', label: b, value: b }))),
        ...(filters.status.map(s => ({ key: 'status', label: s, value: s }))),
        ...(filters.search ? [{ key: 'search', label: `Search: ${filters.search}`, value: filters.search }] : []),
        ...(filters.priceRange[0] > 0 || filters.priceRange[1] < 100000
          ? [{ key: 'priceRange', label: `$${filters.priceRange[0]} - $${filters.priceRange[1]}`, value: filters.priceRange }]
          : []
        ),
      ]

      if (activeFilters.length === 0) return null

      return (
        <div className="flex items-center gap-2 px-6 py-3 border-b bg-muted/20">
          <span className="text-sm text-muted-foreground">Active filters:</span>

          <div className="flex flex-wrap gap-2">
            {activeFilters.map(filter => (
              <Button
                key={`${filter.key}-${filter.value}`}
                variant="secondary"
                size="sm"
                onClick={() => {
                  // Remove filter logic
                  if (filter.key === 'brand') {
                    setFilter('brand', filters.brand.filter(b => b !== filter.value))
                  } else if (filter.key === 'status') {
                    setFilter('status', filters.status.filter(s => s !== filter.value))
                  } else if (filter.key === 'search') {
                    setFilter('search', '')
                  }
                }}
                className="h-7"
              >
                {filter.label}
                <X className="w-3 h-3 ml-1" />
              </Button>
            ))}
          </div>

          <Button
            variant="ghost"
            size="sm"
            onClick={clearAllFilters}
            className="ml-auto"
          >
            Clear all
          </Button>
        </div>
      )
    }
    ```

    **Visual design:**
    - Pills use `variant="secondary"` (subtle background)
    - X icon for removal (clear affordance)
    - "Clear all" button on right (alignment with reading order)
    - Responsive: Wraps on mobile, single row on desktop

    This component sits above DataGrid in catalog page.

  <name>Task 5: Build CommandPalette with Cmd+K shortcut</name>
  <files>apps/web/src/components/layout/CommandPalette.tsx</files>
  <action>
    Create Command Palette component using cmdk for power user search:

    **Features:**
    - Open with Cmd+K (Mac) or Ctrl+K (Windows)
    - Close with Esc
    - Fuzzy search across vehicles
    - Action execution (publish, edit, delete)
    - Keyboard navigation (arrows + enter)

    **Implementation skeleton:**
    ```tsx
    'use client'

    import { useState, useEffect } from 'react'
    import { Command } from 'cmdk'
    {... imports}

    export function CommandPalette() {
      const [open, setOpen] = useState(false)
      const [search, setSearch] = useState('')
      const { data: vehicles } = useVehicles()

      // Keyboard shortcut listener
      useEffect(() => {
        const down = (e: KeyboardEvent) => {
          if ((e.key === 'k' && (e.metaKey || e.ctrlKey))) {
            e.preventDefault()
            setOpen(open => !open)
          }
        }

        document.addEventListener('keydown', down)
        return () => document.removeEventListener('keydown', down)
      }, [])

      // Filter vehicles by search
      const filteredVehicles = useMemo(() => {
        if (!search) return vehicles?.slice(0, 5) || [] // Show 5 recent when empty

        return vehicles?.filter(v =>
          v.title.toLowerCase().includes(search.toLowerCase()) ||
          v.vin?.toLowerCase().includes(search.toLowerCase())
        ).slice(0, 10) || [] // Show 10 results
      }, [vehicles, search])

      return (
        <Command.Dialog open={open} onOpenChange={setOpen}>
          <Command.Content className="bg-background border shadow-lg rounded-lg max-w-md">
            <Command.Input
              placeholder="Search vehicles by VIN, make, model..."
              value={search}
              onValueChange={setSearch}
            />

            <Command.List>
              {filteredVehicles.length === 0 ? (
                <Command.Empty>No vehicles found.</Command.Empty>
              ) : (
                <Command.Group heading="Vehicles">
                  {filteredVehicles.map(vehicle => (
                    <Command.Item
                      key={vehicle.id}
                      onSelect={() => {
                        // Navigate to vehicle detail or execute action
                        window.location.href = `/catalog/${vehicle.id}`
                        setOpen(false)
                      }}
                    >
                      <Car className="w-4 h-4 mr-2" />
                      {vehicle.title}
                      <span className="ml-auto text-xs text-muted-foreground">
                        {vehicle.status}
                      </span>
                    </Command.Item>
                  ))}
                </Command.Group>
              )}

              {/* Actions section */}
              <Command.Group heading="Actions">
                <Command.Item onSelect={() => { /* Publish action */ }}>
                  <Upload className="w-4 h-4 mr-2" />
                  Publish vehicle...
                </Command.Item>
                <Command.Item onSelect={() => { /* Create vehicle */ }}>
                  <Plus className="w-4 h-4 mr-2" />
                  Create new vehicle
                </Command.Item>
              </Command.Group>
            </Command.List>
          </Command.Content>
        </Command.Dialog>
      )
    }
    ```

    **Mobile alternative:**
    - No Cmd+K on mobile (no physical keyboard)
    - Add prominent search button in Header that opens fullscreen overlay
    - Same cmdk component but fullscreen on mobile

    **Accessibility:**
    - Focus trap when open
    - Escape to close
    - ARIA labels for screen readers

    This is a Client Component ('use client') due to interactivity.

  <name>Task 6: Integrate search into catalog page with hybrid filtering</name>
  <files>apps/web/src/app/(seller)/catalog/page.tsx</files>
  <action>
    Update catalog page to integrate all search components with hybrid client/server filtering:

    **Structure:**
    ```tsx
    import { DataGrid } from '@/components/datagrid/DataGrid'
    import { FilterSidebar } from '@/components/filters/FilterSidebar'
    import { FilterPills } from '@/components/filters/FilterPills'
    import { CommandPalette } from '@/components/layout/CommandPalette'
    import { useVehicleFilters } from '@/lib/hooks/useVehicleFilters'
    import { useVehicles } from '@/lib/api/vehicles'
    import { useDeferredValue, useMemo } from 'react'

    export default function CatalogPage() {
      const { filters } = useVehicleFilters()
      const { data: vehicles, isLoading, error } = useVehicles()

      // Client-side instant search (0ms latency)
      const deferredSearch = useDeferredValue(filters.search)
      const filteredVehicles = useMemo(() => {
        if (!deferredSearch) return vehicles

        return vehicles?.filter(v =>
          v.title.toLowerCase().includes(deferredSearch.toLowerCase()) ||
          v.vin?.toLowerCase().includes(deferredSearch.toLowerCase())
        ) || []
      }, [vehicles, deferredSearch])

      // Server-side deep search (called after 300ms debounce)
      // Handled by useVehicleFilters hook URL updates

      return (
        <div className="flex h-[calc(100vh-4rem)]">
          {/* Collapsible filter sidebar */}
          <FilterSidebar />

          {/* Main content */}
          <div className="flex-1 flex flex-col overflow-hidden">
            <div className="p-6">
              <h1 className="text-2xl font-bold mb-4">Catálogo</h1>
            </div>

            {/* Active filter pills */}
            <FilterPills />

            {/* DataGrid with filtered data */}
            <div className="flex-1 overflow-auto px-6">
              <DataGrid data={filteredVehicles} />
            </div>
          </div>

          {/* Command palette (hidden by default) */}
          <CommandPalette />
        </div>
      )
    }
    ```

    **Hybrid search strategy:**
    1. **Client-side (instant):** `useDeferredValue` + `useMemo` for text search on loaded data
       - 0ms latency
       - Feels incredibly fluid
       - Works for ~1000 rows in memory

    2. **Server-side (deep):** URL params trigger API call with filters
       - <200ms response
       - Handles complex queries (price ranges, multi-faceted filters)
       - Scalable to 10,000+ vehicles

    3. **Transition:** After 300ms of idle typing, client search switches to server search
       - Implemented in useVehicleFilters hook with debounce

    **Mobile responsive:**
    - Filter sidebar transforms to drawer (full-screen overlay)
    - DataGrid takes full width when sidebar closed
    - Command palette becomes fullscreen overlay

    **Performance:**
    - `useDeferredValue` prevents search input from blocking UI (React 18+)
    - `useMemo` prevents re-filtering on every render
    - Debounce prevents API hammering

    Update existing catalog/page.tsx with this structure.
