## Arquitectura Generic Catalog (C3 model) — LOCKED:
```
categories(id, name, slug, attribute_schema JSONB, tenant_id)
products(id, name, price, status, category_id, organization_id, tenant_id, attributes JSONB)
vehicles(id, product_id FK → products ON DELETE CASCADE, vin, make, model, year, ...)
```

## Por qué C3:
- Type safety en campos de vehículos (no todo en JSONB)
- Queries eficientes sin parsear JSON
- Escalable a nuevos nichos sin tocar tablas existentes

## Pyright fix:
- `dict[str, object]` → `dict[str, Any]` en todo el domain layer
- `**kwargs` → `**kwargs: Any` en factory methods
- `default_factory=list` → `default_factory=lambda: []` para listas tipadas

## Brain #7 conditions resueltas:
- No CREATE INDEX CONCURRENTLY dentro de Alembic transaction
- Pre-flight test verifica duplicate JSON keys antes de migration
