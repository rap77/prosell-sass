# Plan 12-03 — Vehicle API + Product Hard Delete

## Goal

Create a typed `VehicleResponse` DTO and replace raw `dict` returns in the vehicle router.
Add `GET /vehicles/{id}` endpoint. Add `DELETE /products/{id}` with CASCADE (ON DELETE CASCADE
already in DB — verified in migration `abc123def456`). Implement `DeleteProductUseCase` with
tenant_id enforcement inside the use case (Brain #7 Condition 1).

## Success Criteria

- `POST /api/v1/vehicles` returns typed VehicleResponse (not raw dict)
- `GET /api/v1/vehicles/{vehicle_id}` returns 200 with VehicleResponse or 404
- `GET /api/v1/vehicles/vin/{vin}` returns VehicleResponse (not raw dict)
- `DELETE /api/v1/products/{id}` returns 204 and cascades to vehicle deletion
- `DeleteProductUseCase` verifies tenant_id before deleting (use case invariant, not router)

## Context

**What already exists:**
- `create_vehicle` returns `dict` with 6 fields (gap: not typed, fewer fields than vehicle entity)
- `get_vehicle_by_vin` returns `dict` (gap)
- `get_vehicle_by_product` returns `dict` (gap)
- No `GET /vehicles/{vehicle_id}` by UUID endpoint
- No `DELETE /products/{id}` endpoint (only soft-delete /archive)
- `vehicles.product_id` FK has `ON DELETE CASCADE` — confirmed in migration abc123def456 (line 79)

**Brain #7 Condition C — VERIFIED (delete is hard delete, no issue):**
`SqlAlchemyProductRepository.delete()` does `await self.session.delete(model)` — this is a real
ORM hard delete (generates DELETE SQL), NOT a soft delete. The docstring says "soft delete via
archive" but the implementation is hard delete. CASCADE on `vehicles.product_id` WILL fire when
product is deleted. SC-6 is NOT broken. No `hard_delete()` method needed — `delete()` is already
permanent. The docstring mismatch should be corrected in the implementation (task 12-03-05 scope).

```python
"""Delete product use case — hard delete with CASCADE."""

import logging
from uuid import UUID

from prosell.domain.repositories.product_repository import AbstractProductRepository

logger = logging.getLogger(__name__)


class DeleteProductUseCase:
    """
    Hard-delete a product by ID.
    
    ON DELETE CASCADE on vehicles.product_id means the vehicle record
    is automatically deleted by PostgreSQL when the product is deleted.
    
    tenant_id is verified INSIDE this use case — not delegated to the router.
    """

    def __init__(self, product_repository: AbstractProductRepository) -> None:
        self.product_repository = product_repository

    async def execute(self, product_id: UUID, tenant_id: UUID) -> None:
        """
        Delete product and cascade to vehicle.
        
        Args:
            product_id: UUID of product to delete
            tenant_id: Tenant ID from authenticated user (used for access control)
            
        Raises:
            ValueError: If product not found OR product belongs to different tenant
        """
        # tenant_id enforced here: get_by_id filters by both id AND tenant_id
        product = await self.product_repository.get_by_id(product_id, tenant_id)
        if not product:
            raise ValueError(f"Product {product_id} not found or access denied")
        
        logger.info(
            "Hard-deleting product %s (tenant %s) — vehicle cascade will auto-run",
            product_id,
            tenant_id,
        )
        
        await self.product_repository.delete(product_id, tenant_id)
```
