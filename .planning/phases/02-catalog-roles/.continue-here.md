<decisions_made>

**Approach: Mock the User entity directly**

Instead of fighting pytest fixture injection, create a simple mock User that has `has_role()` return True:

```python
from unittest.mock import Mock
mock_user = Mock(spec=User)
mock_user.id = admin_user.id
mock_user.has_role = Mock(return_value=True)  # Always True for admin tests
```

This avoids the `_roles` attribute issue entirely and is simpler for integration tests.
</decisions_made>

<blockers>

**Pytest Fixture Injection Issue**

The problem: FastAPI's `Depends()` calls the dependency function, but pytest fixtures create new instances each time. The `admin_user._roles = [admin_role]` assignment doesn't persist because:

1. The fixture returns a User instance
2. get_mock_user() creates a NEW User instance (copying fields)
3. The new instance doesn't have _roles set

**Workaround:** Use Mock objects instead of real User entities in test dependencies.
</blockers>
