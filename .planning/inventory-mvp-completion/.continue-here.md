<decisions_made>

**Root Cause Analysis (React Compiler Warnings):**
- **Problem**: `setIsClient(true)` in useEffect caused double-rendering
- **Why obsolete**: Next.js 16 + React 19 don't need `isClient` pattern
- **Solution**: Use `isLoading` from store directly
- **Reference**: [React.dev: set-state-in-effect](https://react.dev/learn/you-might-not-need-an-effect)

**Key Learning:**
ESLint/React Compiler warnings are rarely false positives. The "client-side only" pattern with `isClient` state is from Next.js 12 era and is no longer needed.
</decisions_made>
